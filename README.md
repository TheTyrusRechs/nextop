# Next Op ‚Äî Shop Floor Queue

Single-file dark-themed web app for shop floor operators to pick an operation and see the prioritized job queue. Includes a planner-only admin mode (PIN-protected) for marking jobs HOT and viewing scheduled completion ETAs.

Data is pulled in real time from an Epicor Kinetic BAQ via a server-side nginx reverse proxy, so credentials never reach the browser.

Default admin PIN: **`2580`** (change before deploying ‚Äî search for `DEFAULT_PIN` in `public/index.html`).

## Architecture

```
 Browser  ‚îÄ‚îÄ>  /api/jobs  ‚îÄ‚îÄ[nginx proxy]‚îÄ‚îÄ>  Epicor BAQ
                              ^
                              ‚îÇ injects Basic Auth header
                              ‚îÇ from EPICOR_BASIC_AUTH env var
                              ‚îÇ uses URL from EPICOR_URL env var
```

The browser sees only same-origin JSON. Credentials live in App Service environment variables, never in the image, never in client code.

---

## Required environment variables

Both must be set in App Service ‚Üí Settings ‚Üí Environment variables (and locally in a `.env` file for `docker compose`):

| Name | Value |
|---|---|
| `EPICOR_URL` | Full BAQ URL including the `?api-key=...` query string. Example: `https://<host>/<instance>/api/v2/odata/<company>/BaqSvc/<BAQName>/Data?api-key=YOURKEY` |
| `EPICOR_BASIC_AUTH` | Base64 of `username:password`. Generate with: `echo -n 'youruser:yourpass' \| base64` |
| `WEBSITES_PORT` | `8080` (App Service only ‚Äî tells the platform which port the container listens on) |

## Deploying to Azure

This repo is set up to deploy as a **Linux Container** App Service on the **Basic B1** tier (~$13/mo) with images pushed to **Azure Container Registry** (Basic tier, ~$5/mo). You only set this up once.

1. Create resource group, ACR (admin user enabled), and App Service (Linux Container, B1) via the Azure Portal.
2. Open App Service ‚Üí **Deployment Center** ‚Üí Source: GitHub ‚Üí select this repo and branch ‚Üí Registry: ACR ‚Üí Image name: `nextop`. Save. This auto-generates `.github/workflows/main_<app-name>.yml` with the right secrets ‚Äî don't edit it.
3. Open App Service ‚Üí **Environment variables** ‚Üí add `WEBSITES_PORT=8080`, `EPICOR_URL=...`, `EPICOR_BASIC_AUTH=...`. Apply.
4. Push to GitHub ‚Üí workflow runs ‚Üí image lands in ACR ‚Üí App Service pulls it. ~3 min end to end.

After the first successful deploy, every push to your main branch redeploys automatically.

## Updating the Epicor API key

If the key is rotated or compromised:
1. Generate a new key in Epicor (System Setup ‚Üí Security ‚Üí API Key Maintenance).
2. Update `EPICOR_URL` in App Service ‚Üí Environment variables ‚Üí save.
3. App Service auto-restarts the container; the new key is picked up at startup.

No code changes or redeploys needed.

## Field mapping (Epicor BAQ ‚Üí app)

The BAQ `MO-OpenJobNextOp` returns these columns; the app remaps them in `transformEpicorRecord()` (top of the script in `public/index.html`):

## Refresh behavior

- Page load: fetch `/api/jobs`, render
- Auto-refresh: every 5 minutes, silently refetch and re-render
- Manual: page reload

The nginx proxy does not cache ‚Äî every browser fetch hits Epicor. With ~5-min auto-refresh and ~10 tablets on the floor, that's ~120 BAQ runs/hour, which is fine for Kinetic SaaS.
